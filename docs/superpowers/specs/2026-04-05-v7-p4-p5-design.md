## Context

V7 P1-P3 (Tap firehose, PDS setup, private network) and P6-P9 (hook pipeline, declarative indexers, scripting engine, MCP) are merged to main. P4 and P5 are the remaining phases.

**P4 was originally "Deploy Ozone container"** but Ozone is heavyweight moderation infrastructure designed for Bluesky-scale content moderation. For our governance labels (6 label types, cooperative-scoped), implementing the ATProto labeler protocol natively is simpler: expose `com.atproto.label.queryLabels` (REST) and `com.atproto.label.subscribeLabels` (WebSocket) endpoints from our existing `governance_label` table.

**P5 cleanup is scoped by P6**: The `pds_record` table was originally V3 simulation but P6's hook pipeline repurposed it as the source of truth. Only `pds_commit`, `federation_peer`, `federation_outbox`, and `plc_operation` are truly dead.

## Phase P4: Native ATProto Label Service

### Goal

Make governance labels visible across the ATProto network by implementing the standard labeler protocol natively on our AppView.

### Architecture

```
GovernanceLabeler.emitLabel()
  |
  |-- INSERT into governance_label (with seq)
  |-- LabelSigner.sign(label) -> sig bytes
  |-- LabelSubscriptionManager.notify(label)
        |
        |-- Emit to connected WebSocket clients (DAG-CBOR frames)
        |
  REST queryable via /xrpc/com.atproto.label.queryLabels
```

### Existing Infrastructure

- `governance_label` table (migration 031): id, src_did, subject_uri, subject_cid, label_value, neg, created_at
- `GovernanceLabeler` service with emitLabel/negateLabel/getLabelsForSubject/queryLabels
- `ProposalService` emits labels on proposal resolution (lines 486-499)
- `GET /api/v1/labels` REST endpoint (routes/labels.ts)
- `ws@8.20.0` already in API deps
- `@ipld/dag-cbor@9.2.5` in federation package
- `@atproto/crypto` for signing
- Admin UI tabs: pipeline, lexicons, scripts (P6-P9)

### Implementation

#### 1. Migration `055_label_subscription.ts`

Add `seq BIGSERIAL` to `governance_label` for cursor-based subscription replay:

- Column: `seq BIGSERIAL NOT NULL`
- Index: `idx_governance_label_seq` on `(seq)`
- Index: `idx_governance_label_src_did` on `(src_did)` (for `sources[]` filter)

Schema type update in `packages/db/src/schema.ts`: add `seq: Generated<number>` to `GovernanceLabelTable`.

#### 2. Label Subscription Manager (`apps/api/src/services/label-subscription.ts`)

In-process EventEmitter bridging GovernanceLabeler writes to WebSocket clients:

- `notifyNewLabel(label)` â€” emits to connected subscribers
- `handleSubscription(ws, cursor?)` â€” replay from cursor then real-time
- DAG-CBOR frame encoding: header `{ op: 1, t: '#labels' }` + body `{ seq, labels }` concatenated as two CBOR values per the ATProto subscription protocol
- Max 100 concurrent WebSocket clients (configurable)
- 60-second keepalive ping; disconnect idle clients after 5 minutes

DAG-CBOR encoding (add `@ipld/dag-cbor` to `apps/api/package.json`):
```typescript
import * as dagCbor from '@ipld/dag-cbor';
const headerBytes = dagCbor.encode({ op: 1, t: '#labels' });
const bodyBytes = dagCbor.encode({ seq, labels });
// Concatenate into single frame
```

#### 3. Label Signer (`apps/api/src/services/label-signer.ts`)

Signs labels with the cooperative's secp256k1 key for ATProto authenticity:

1. Construct label object without `sig`
2. Encode to DAG-CBOR
3. Sign with `@atproto/crypto` `Secp256k1Keypair` (same pattern as `packages/federation/src/local/plc-signing.ts`)
4. Attach raw signature bytes as `sig`

Key source: `COOP_ROTATION_KEY_HEX` env var (same as PLC operations). Optional â€” labels emitted unsigned when key not configured.

#### 4. GovernanceLabeler Changes (`apps/api/src/services/governance-labeler.ts`)

Extend constructor to accept optional `LabelSubscriptionManager` and `LabelSigner`:

```typescript
constructor(
  private db: Kysely<Database>,
  private subscriptionManager?: LabelSubscriptionManager,
  private labelSigner?: LabelSigner,
)
```

In `emitLabel()` and `negateLabel()`:
- Use `.returning(['seq', 'created_at'])` to capture auto-assigned sequence
- Sign the label if signer is available
- Notify subscription manager with signed label + seq

#### 5. XRPC Routes (`apps/api/src/routes/xrpc-labels.ts`)

**`GET /xrpc/com.atproto.label.queryLabels`**

Query params: `uriPatterns[]`, `sources[]`, `limit` (default 50, max 250), `cursor` (seq-based)

Response:
```json
{
  "cursor": "123",
  "labels": [{ "ver": 1, "src": "did:plc:...", "uri": "at://...", "val": "proposal-approved", "neg": false, "cts": "2026-04-05T12:00:00Z", "sig": "<base64>" }]
}
```

Implementation: query `governance_label` with filters. `uriPatterns` wildcards use SQL `LIKE` (`*` -> `%`). Cursor: `seq > cursor ORDER BY seq ASC`.

**WebSocket `/xrpc/com.atproto.label.subscribeLabels`**

Setup: `WebSocketServer` with `noServer: true`, handle HTTP upgrade for this path only.

Query param: `cursor` (optional integer for replay).

Delegates to `LabelSubscriptionManager.handleSubscription(ws, cursor)`.

#### 6. WebSocket Setup (`apps/api/src/index.ts`)

After `server = app.listen(...)`:
```typescript
import { setupLabelWebSocket } from './routes/xrpc-labels.js';
setupLabelWebSocket(server, container.labelSubscriptionManager);
```

Handles `'upgrade'` event on raw HTTP server, scoped to `/xrpc/com.atproto.label.subscribeLabels` path only.

#### 7. DI Container (`apps/api/src/container.ts`)

Add to Container interface:
- `labelSubscriptionManager: LabelSubscriptionManager`
- `labelSigner: LabelSigner | undefined`

Instantiate before GovernanceLabeler:
```typescript
const labelSubscriptionManager = new LabelSubscriptionManager(db);
const labelSigner = config.COOP_ROTATION_KEY_HEX ? new LabelSigner(config.COOP_ROTATION_KEY_HEX) : undefined;
const governanceLabeler = new GovernanceLabeler(db, labelSubscriptionManager, labelSigner);
```

#### 8. DID Document Labeler Entry

Modify `PlcClient.create()` and `LocalPlcClient.create()` to include optional `atproto_labeler` service entry:

```typescript
services: {
  atproto_pds: { type: 'AtprotoPersonalDataServer', endpoint: params.pdsUrl },
  atproto_labeler: { type: 'AtprotoLabeler', endpoint: params.labelerUrl },
}
```

Add optional `labelerUrl` to `PlcCreateParams`. When set, cooperative's DID document advertises it as a labeler, making labels discoverable across the ATProto network.

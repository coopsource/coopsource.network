# V8.11 — Entity Editing Extended Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend V8.10's entity editing patterns to 6 more entities — Officer, Compliance Item, Fiscal Period, Commerce Listing, Commerce Need, and Member Roles.

**Architecture:** Four sub-phases. V8.11a extracts the admin page's tabs into separate components (prerequisite refactor). V8.11b adds PUT endpoints + edit modals for 3 admin entities. All frontend editing uses the V8.10 modal-reuse pattern (`editingEntity` state + `{#key}` block).

**Tech Stack:** SvelteKit 2, Svelte 5 runes, Tailwind CSS 4, Express 5, Kysely, Vitest, Playwright

- [ ] **Step 2: Add service update methods**

Add `updateOfficer()` to `officer-record-service.ts`:
```typescript
async updateOfficer(id: string, cooperativeDid: string, data: { title?: string; termEndsAt?: string; responsibilities?: string }): Promise<OfficerRow> {
  const [existing] = await this.db.selectFrom('officer_record').selectAll().where('id', '=', id).where('cooperative_did', '=', cooperativeDid).execute();
  if (!existing) throw new NotFoundError('Officer not found');
  if (existing.status !== 'active') throw new ValidationError('Can only update active officers');

  const updates: Record<string, unknown> = {};
  if (data.title !== undefined) updates.title = data.title;
  if (data.termEndsAt !== undefined) updates.term_ends_at = data.termEndsAt ? new Date(data.termEndsAt) : null;
  if (data.responsibilities !== undefined) updates.responsibilities = data.responsibilities || null;

  const [row] = await this.db.updateTable('officer_record').set(updates).where('id', '=', id).returningAll().execute();
  return row;
}
```

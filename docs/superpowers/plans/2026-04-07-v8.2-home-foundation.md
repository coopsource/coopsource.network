# V8.2 — Home Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `/dashboard` with `/me` (the new "Home" workspace), add a workspace switcher to the sidebar, and update login/register/coop-creation redirects.

**Architecture:** Introduce a new workspace type `'home'` that uses the same `AppShell` + `Sidebar` pattern as cooperatives but with a different sidebar mode. Add a `WorkspaceSwitcher` component that replaces the static sidebar header label with a clickable dropdown listing Home + all coops the user belongs to. Move the existing Profile page out of the coop nesting (`/coop/[handle]/profile`) into the user-scoped Home workspace (`/me/profile`) since profile data is user-scoped, not coop-scoped.

**Tech Stack:** SvelteKit 2, Svelte 5 runes, existing AppShell/Sidebar components, no new database tables

**Spec:** `ARCHITECTURE-V8.md` (project root) — Phase V8.2 section

## Context

V8.1 (visibility foundation) is merged. V8.2 fixes the inconsistency the user noticed: after login, the user lands on `/dashboard` (a custom minimal layout, no sidebar). Once they enter a cooperative, they get the new 3-section sidebar. The transition is jarring and there's no path back from a coop to the dashboard.

V8.2 makes the post-login experience use the same `AppShell + Sidebar` as the cooperative workspaces, just with a different sidebar mode ("Home"). The sidebar header becomes a workspace switcher (Slack-style dropdown) so users can navigate between Home and any coop they belong to.

**Scope is intentionally narrow.** V8.2 ships:
- The Home workspace (`/me`) using `AppShell`
- A simple Home page that re-uses the existing dashboard cards (My Coops, My Networks, Pending Invitations)
- A workspace switcher in the sidebar header
- Updated login/register/coop-creation redirects
- Profile page moved from coop to Home workspace

V8.2 does NOT ship:
- Hardcoded widget framework or `home_layout` table (deferred — V8.2 just shows static cards)
- Explore/Matches sidebar items (deferred to V8.4/V8.7 when those features exist)
- Per-profile Home configuration (V8.3 introduces profiles)
- True customization UI (deferred to V8.X)

Create `apps/web/src/lib/actions/click-outside.ts`:

```typescript
import type { Action } from 'svelte/action';

/**
 * Svelte action: fires the callback when a click happens outside the element.
 *
 * Usage: <div use:clickOutside={() => (open = false)}>
 *
 * The listener is attached on mount and cleaned up on destroy. Uses capture
 * phase (third arg `true`) so the handler runs before bubbling click handlers,
 * which prevents timing issues where the outside-click fires before inner
 * elements' click handlers have a chance to run.
 */
export const clickOutside: Action<HTMLElement, () => void> = (node, callback) => {
  function handle(event: MouseEvent) {
    if (!node.contains(event.target as Node)) {
      callback();
    }
  }

  // Attach on next tick so the click that OPENED the element doesn't
  // immediately trigger the outside-click handler.
  let mounted = false;
  queueMicrotask(() => {
    mounted = true;
    document.addEventListener('click', handle, true);
  });

  return {
    destroy() {
      if (mounted) {
        document.removeEventListener('click', handle, true);
      }
    },
  };
};
```

```typescript
import type { Component } from 'svelte';
import Users from '@lucide/svelte/icons/users';
import Vote from '@lucide/svelte/icons/vote';
import MessageSquare from '@lucide/svelte/icons/message-square';
import Compass from '@lucide/svelte/icons/compass';
import Banknote from '@lucide/svelte/icons/banknote';
import Globe from '@lucide/svelte/icons/globe';
import Shield from '@lucide/svelte/icons/shield';
import DollarSign from '@lucide/svelte/icons/dollar-sign';
import Handshake from '@lucide/svelte/icons/handshake';
import Settings from '@lucide/svelte/icons/settings';
import CircleUser from '@lucide/svelte/icons/circle-user';
import type { WorkspaceContext } from '$lib/api/types.js';

export interface NavItem {
  href: string;
  label: string;
  icon: Component;
}

export interface NavSection {
  label: string; // empty string = no section header rendered
  items: NavItem[];
}

/** Map of sub-routes → parent sidebar item, for active state highlighting. */
export const ROUTE_PARENT_MAP: Record<string, string> = {
  '/invitations': '/members',
  '/onboarding': '/members',
  '/agreements': '/governance',
  '/legal': '/governance',
  '/agents': '/admin',
  '/notifications': '/settings',
  '/settings/connections': '/settings',
  '/settings/payments': '/settings',
};

/** Is the user's role set elevated to officer/admin/owner? */
export function isAdminRoles(roles: readonly string[] | undefined): boolean {
  return roles?.some((r) => ['admin', 'owner', 'officer'].includes(r)) ?? false;
}

/** Compute the Cooperative nav section for a coop workspace. */
export function cooperativeNavSection(
  workspace: WorkspaceContext | null | undefined,
  isAdmin: boolean,
): NavSection {
  const prefix = workspace?.prefix ?? '';
  if (!prefix || workspace?.type !== 'coop') return { label: 'Cooperative', items: [] };

  const isNetwork = workspace?.cooperative?.isNetwork ?? false;
  const membersLabel = isNetwork ? 'Cooperatives' : 'Members';

  const items: NavItem[] = [
    { href: `${prefix}/members`, label: membersLabel, icon: Users },
    { href: `${prefix}/governance`, label: 'Governance', icon: Vote },
    { href: `${prefix}/posts`, label: 'Posts', icon: MessageSquare },
  ];

  if (isAdmin) {
    items.push({ href: `${prefix}/finance`, label: 'Finance', icon: DollarSign });
    items.push({ href: `${prefix}/admin`, label: 'Admin', icon: Shield });
  }

  return { label: 'Cooperative', items };
}

/** Compute the Network nav section for a coop workspace. */
export function networkNavSection(
  workspace: WorkspaceContext | null | undefined,
): NavSection {
  const prefix = workspace?.prefix ?? '';
  if (!prefix || workspace?.type !== 'coop') return { label: 'Network', items: [] };

  return {
    label: 'Network',
    items: [
      { href: `${prefix}/networks`, label: 'Networks', icon: Globe },
      { href: `${prefix}/partners`, label: 'Partners', icon: Handshake },
      { href: `${prefix}/alignment`, label: 'Alignment', icon: Compass },
      { href: `${prefix}/campaigns`, label: 'Campaigns', icon: Banknote },
    ],
  };
}

/** Compute the You nav section (user-scoped items). */
export function youNavSection(
  workspace: WorkspaceContext | null | undefined,
): NavSection {
  // Profile is always user-scoped: /me/profile
  // Settings is context-scoped: /me/settings in home mode, /coop/X/settings in coop mode
  const isHome = workspace?.type === 'home';
  const settingsHref = isHome
    ? '/me/settings'
    : workspace?.prefix
      ? `${workspace.prefix}/settings`
      : '/me/settings';

  return {
    label: 'You',
    items: [
      { href: '/me/profile', label: 'Profile', icon: CircleUser },
      { href: settingsHref, label: 'Settings', icon: Settings },
    ],
  };
}

/** Check whether a sidebar item href is active for the current pathname. */
export function isNavItemActive(
  href: string,
  pathname: string,
  workspacePrefix: string | undefined,
): boolean {
  if (pathname === href || pathname.startsWith(href + '/')) return true;

  if (workspacePrefix) {
    for (const [subRoute, parentRoute] of Object.entries(ROUTE_PARENT_MAP)) {
      const fullSub = workspacePrefix + subRoute;
      const fullParent = workspacePrefix + parentRoute;
      if (href === fullParent && (pathname === fullSub || pathname.startsWith(fullSub + '/'))) {
        return true;
      }
    }
  }

  return false;
}
```

```typescript
import { error } from '@sveltejs/kit';
import { createApiClient } from '$lib/api/client.js';
import type { LayoutServerLoad } from './$types.js';

export const load: LayoutServerLoad = async ({ params, fetch, request, locals }) => {
  const cookie = request.headers.get('cookie') ?? undefined;
  const api = createApiClient(fetch, cookie);

  try {
    const cooperative = await api.getCooperativeByHandle(params.handle);

    // Fetch user's roles for THIS cooperative (V8.1)
    let userRoles: string[] = [];
    if (locals.user) {
      try {
        const member = await api.getMember(locals.user.did);
        userRoles = member.roles;
      } catch {
        // User may not be a member of this cooperative
      }
    }

    // V8.2 — Fetch myCoops for the workspace switcher dropdown.
    let myCoops: Awaited<ReturnType<typeof api.getMyMemberships>>['cooperatives'] = [];
    let myNetworks: typeof myCoops = [];
    try {
      const result = await api.getMyMemberships();
      myCoops = result.cooperatives;
      myNetworks = result.networks;
    } catch {
      // empty arrays
    }

    return {
      workspace: {
        type: 'coop' as const,
        handle: params.handle,
        prefix: `/coop/${params.handle}`,
        cooperative,
        userRoles,
      },
      myCoops: [...myCoops, ...myNetworks],
    };
  } catch {
    error(404, 'Cooperative not found');
  }
};
```

## Design Decisions (resolved during planning review)

1. **Sidebar refactor**: Task 2a extracts nav config into `sidebar-nav.ts` as pure functions. Task 2b then adds Home mode using the same pattern. This keeps the Sidebar component focused on rendering while the nav config is testable and maintainable.

2. **Profile page simplified**: Dropped the "My Activity" and "My Cooperatives" tabs (both duplicate content shown on /me and in the sidebar). Profile is now a single page showing user identity (displayName, handle, email, DID). Multi-profile tabs come back in V8.3.

3. **Header = dropdown trigger only**: Clicking the workspace switcher header opens the dropdown. An "Activity" sidebar item provides a direct navigation link back to /me. This matches Slack/Linear/Notion convention.

4. **Settings is a real minimal page**: /me/settings shows theme toggle and read-only account info. Not a stub.

5. **clickOutside via Svelte action**: WorkspaceSwitcher uses a reusable `use:clickOutside` action (defined in `$lib/actions/click-outside.ts`) with `queueMicrotask` delay to prevent the opening click from immediately closing the dropdown.

## Risk Mitigations

1. **Cooperative loaded with `null` cooperative field**: After Task 1, `WorkspaceContext.cooperative` is nullable. Any existing code that reads `workspace.cooperative.X` without optional chaining will break. The build step in Task 1 will surface these. Fix by adding `?.` everywhere needed. Grep `workspace\.cooperative\.` in `apps/web/src` to pre-emptively find them.

2. **myCoops loaded three times**: V8.2 loads `myCoops` in `/me/+layout.server.ts`, `/coop/[handle]/+layout.server.ts`, and `/net/[handle]/+layout.server.ts`. This is fine — each layout is independent and the data is needed for the workspace switcher. SvelteKit's parent data inheritance means child pages (like `/me/profile`) don't need to re-fetch — they read `data.myCoops` from the parent layout data.

3. **WorkspaceSwitcher dropdown with many coops**: For users belonging to 10+ coops, the dropdown becomes long. V8.2 doesn't add filtering — that's a future enhancement. The dropdown has `overflow-y: auto` via CSS if needed (not currently set — add if user testing shows it's needed).

4. **/coop/[handle]/profile breaks bookmarks**: Users who bookmarked the old profile URL will get a 404. This is a deliberate trade-off — the page existed for ~1 day on main (added in V7 sidebar redesign, deleted in V8.2). If this becomes an issue later, we can add a redirect.

5. **Sidebar header label fallback**: If a user lands on a route without a workspace context, the AppShell falls back to "Co-op Source". This is fine because layout servers always run before page servers in SvelteKit, and every authed route has a layout that sets workspace.

6. **Existing E2E tests reference /dashboard**: Multiple tests across the suite reference `/dashboard`. Task 5 covers `auth.spec.ts` and `dashboard.spec.ts` explicitly; if other tests break, fix them as part of Task 5. Pre-existing `.fixme` tests get their target URLs updated to `/me` even while remaining skipped.

7. **WorkspaceSwitcher collapsed mode**: When the sidebar is collapsed (56px wide), the switcher shows only a Home or Building2 icon. The dropdown is 224px wide and extends to the right of the collapsed sidebar. This works but should be visually verified during manual testing.
